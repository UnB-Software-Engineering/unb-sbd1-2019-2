Exercício de Revisão para P1
PROBLEMA

O principal cargo político nacional, Presidente da República, terá o seu processo eleitoral diferenciado em uma implementação nova para acompanhar as próximas eleições. Um candidato a presidente terá armazenado o título de eleitor de 12 números, a data da eleição e o número de candidato (inteiro com 2 dígitos), sendo único para cada candidato em uma mesma eleição. O eleitor será identificado pelo título de eleitor, data de nascimento, nome completo, e idade em anos já completos.  O registro de um voto exigirá o título do eleitor e a data da eleição. Cada voto é especializado em Justificativa ou Efetivação do voto, em que cada eleitor só poderá votar uma vez em um candidato a presidente na mesma eleição, mas a base de dados deverá ser elaborada para guardar os votos de várias eleições para presidente. A Justificativa consiste na participação do eleitor na eleição, mas ele não vota em ninguém e deverá ser armazenado um identificador na justificativa que identifique o eleitor e a eleição que ele está justificando, além da descrição da justificativa com até 1000 caracteres. O processo de Efetivação do voto na eleição armazenará o identificador do eleitor e a data da eleição, além do número único que identifica o candidato para presidente em uma eleição específica.


Baseado no Problema apresentado acima, deverá ser elaborado o Modelo Entidade-Relacionamento (ME-R) que iniciará o seu Projeto de Banco de Dados mais coerente ao nível Conceitual para a sua solução. Não esqueça de identificar corretamente cada elemento relevante para o ME-R, como vem sendo feito em sala de aula, de maneira coerente e completa ao ME-R.

Em seguida, elabore o Diagrama Entidade-Relacionamento (DE-R) completo, e totalmente coerente com o ME-R desenvolvido na sua proposta do Projeto de Banco de Dados que soluciona, completamente, o Problema indicado nesta atividade usando o brModelo 3.2. Atente ao ME-R elaborado, pois se ele não estiver completo ou incorreto resultará em erro grave que comprometerá todas as demais solicitações desta atividade, pois existirá um erro na parte inicial do seu projeto de banco de dados.

Respeitando as definições do ME-R e do DE-R da sua solução proposta para o problema acima, elabore ainda o Diagrama Lógico (DL) no brModelo 3.2, correspondente aos conteúdos anteriores que resultarão no desenvolvimento do Projeto de Banco de Dados, que deverá solucionar o problema proposto no enunciado acima.

Condizente com todas as definições existentes em sua proposta de projeto de banco de dados, elaboradas anteriormente (ME-R, DE-R e DL) para solucionar esta atividade, deverão também ser criados três scripts para o projeto proposto, onde o primeiro cria a base de dados (database), tendo como nome seu primeiro e último nome concatenados, sem espaço e acentos. Dentro desta base de dados no MySQL o primeiro script deverá criar todas as tabelas existentes em seu projeto, respeitando todas as regras, definições e restrições até a terceira forma normal (3FN) estudadas e que deverão existir em seu projeto que soluciona o Problema acima proposto.

O segundo script deverá inserir pelo menos três tuplas em cada tabela existente em seu projeto proposto como a solução do Problema. É importante esclarecer que a não execução completa dos scripts, ou se ele resultar em armazenamento de dados (tuplas) incoerentes, que não respeitam as definições e restrições indicadas em seu projeto de solução, a nota relacionado aos scripts será ZERO na atividade, similar as exigências da P1, quando solicita a elaboração de scripts. Os dados que alimentarem a base de dados deverão corresponder aos dados reais em todas as tuplas e seus atributos, simulando, exatamente, o que poderia estar armazenado na base dados elaborada.

O terceiro script deverá apagar todas as tabelas do projeto, sem apagar a base de dados. Por isso também é importante a existência no primeiro script da instrução que verifica a existência ou não da base de dados e só cria a base de dados se ela NÃO existir, pois se o primeiro script for executado novamente depois de todas as tabelas serem apagadas o script não apresentará erro e criará todas as tabelas corretamente na base de dados já existente.
